PSD Cases area

Pages: Your cases, Team cases and All cases - Search: 3x page designs showing two new pages with all 3 incorporating an additional left-hand sub-nav.

  • When the user selects the "Cases" option from the top level navigation, the user now lands on the "Your cases" page. The parent nav "Cases" active state remains active for all 3 pages.
  • When the user selects the "Your cases" link from the Home page, the user now lands on the new "Your cases" page.
  • When the user selects the "All cases" link from the Home page, the user lands on the "All cases" page.

1. Your cases:

  • The page automatically loads displaying all the open cases owned by the user
  • The list of open cases are automatically ordered by most recently created
  • If there are 12 or more cases. The 'Sort by' option is added to the page.
    (There is a new sorting option of Oldest cases)
  • If there are no (zero) cases, the sorting options are omitted
  • The center table column now displays the case created date
  • As before: Pagination for 20 or more cases. Table footer included for 12 or more cases. (New ID added to content area and removed from parent table tag on this page).
  1. Your cases page (without sorting)
  2. Your cases page (with sorting)
  3. Your cases page (with no cases)

2. Team cases: (functionality as 'Your cases')

  • The page automatically loads displaying all the open cases owned by the team
  • The list of open cases are automatically ordered by most recently created
  • If there are 12 or more cases. The 'Sort by' option is added to the page.
    (There is a new sorting option of Oldest cases)
  • If there are no (zero) cases, the sorting options are omitted
  • The center table column now displays the case created date
  • As before: Pagination for 20 or more cases. Table footer included for 12 or more cases. (New ID added to content area and removed from parent table tag on this page).
  1. Team cases page (without sorting): see 'Your cases' example
  2. Team cases page (with sorting)
  3. Team cases page (with no cases)

3. All cases - Search:

  • No change to the current functionality
  • Add 'Oldest cases' to the sorting options to match the two new pages
  • There is new content for the drop-down help text. The second paragraph (spreadsheet advice) is contextual, as like the download link elsewhere in the page
  • We agreed in refinement that the persisted filters behaviour would be removed, so that if and when a user changes the filters but then navigates away and then navigates back to the 'All cases - Search' page, the filters will have returned to their default settings. (Filter changes only persist while the user is interacting with the page).
  • Design changes:
    • Updates to the H1 title and 'help' link text
    • New main hint text below the H1
    • Keywords input field moved to new row, margin/padding and input width classes changed
    • Additional ID "page-content" added to main section (for skip nav)
    • Changes to the classes/styling on the 'Filters' header and the legends for the filter groups
    • Changed the search input label to just "Search" and changed the hidden text on the search button to "Submit search".
  1. All cases - Search.

Dev notes: Sub-nav has 'aria-current' on active hyperlink. You'll need the latest _opss-psd.scss file.
