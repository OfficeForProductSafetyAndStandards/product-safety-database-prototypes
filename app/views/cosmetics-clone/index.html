  2. What is the product name?
    Error message: "Enter the product name" (if left blank). The template source product is name-checked 3 times). 'Cancel' returns the user to the previous page (as does the 'Back' link).
    • What is the product name? - an alternative design
  3. Draft created confirmation
    With links to the cosmetic products page and to the draft via the task list page. We "clone" the notification including labels but we do not clone/include ingredient PDFs.
  4. Draft opened in the task page
    This typical example illustrates how the draft is edited in the usual way. (The H1 is populated with the new product name). All the established behaviour of how the tasks page works is followed,* allowing a user to work their way through all the tasks.
    • Each page in each task is pre-populated with data from the original notification; except potentially for ingredients, but this scenario is already dealt with by the tasks process when opening an old draft (with ingredients PDF) in the new ingredients manual input version of the service (where the task journey will automatically request ingredient information)
    • It is important that the user 'clicks' through each task, so no task has the status of 'Completed' when the draft is created. It is noted that zero tasks have been completed in the sub-header: e.g, "Incomplete: 0 of 4 tasks have been completed."
    • The final 'accept and submit' task has the status of 'Cannot start yet' until all the previous tasks have been 'clicked' through and their status becomes 'Completed'
    • There is new content for the right-hand column, which is the only right-hand column content used for drafts which were 'cloned' (templated drafts)
    • *Essentially this scenario is best described as being a normal draft which has completed all the tasks – except for the final 'accept and submit' task – except that the status on all the other tasks is not shown as 'Completed' but as 'Not started' instead. When each task has been 'clicked' through in full their status changes to 'Completed', but as like a 'normal' draft the user can click through these tasks in any order (except for the final task which must be completed last and only changes to 'Not started' once all the other tasks are 'Completed').
