Moved hint text with updated classes. Remove H2. Remove "of your notified products" text from notified products download link. Remove the two margin classes from the download link).    .
    • New left nav - extended
      Provides example of how the left nav will/can eventually be used.
  2. Drafts page
    Maintain the same functionality - or lack of it - and simply split out the drafts information into its own page. (Any embellishments for the drafts page is a seperate piece of work).
  3. Product - search
    Search form. (Update "aria-label" on the section tag). The main search box returns fuzzy matches on the product name, but if a UKCP number is entered we should search for "UKCP-61869641" or "61869641". There is no error message required on the search input (as a it can be blank). Both the blue button and the 'search' button submit the form.

    The filtering on notified or archived notifications is shown for illustrative purposes - once archiving is introduced we should include this filter.

    The reset link is a standard reset (resetting the form); there is no need for hidden inputs (which won't need resetting).
    • Product - search - errors
      Example of date form errors. I recommend only using these two date error messages:
      1. Enter a value for all date fields or none. ....(if one date field has a date, they all need a value)
      2. Enter only valid dates.      . ....(numbers which are not valid dates and non-numeric values)

    Clicking 'Search again' returns the user back to the search page (and the search page form is blank).
